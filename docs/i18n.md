# Internationalization

AiControlCenter ships with 5 UI locales:

| Locale | File |
| --- | --- |
| English (source) | `i18n/en.json` |
| Português (BR) | `i18n/pt_BR.json` |
| 简体中文 | `i18n/zh_CN.json` |
| Español | `i18n/es_ES.json` |
| Deutsch | `i18n/de_DE.json` |

## Locale resolution

1. If **Language** is set to anything other than `auto`, that locale wins.
2. Otherwise the system locale (`Qt.locale().name`) is normalized.
3. Anything unmapped falls back to English (`en.json`).
4. Missing keys in an active locale fall back to the English bundle.

## Contributing translations

All bundles must keep **key parity** with `en.json` (enforced in CI). Edit the
relevant `i18n/*.json` and run the parity check:

```bash
python3 - <<'PY'
import json
from pathlib import Path
base = set(json.loads(Path('i18n/en.json').read_text()).keys())
for p in Path('i18n').glob('*.json'):
    keys = set(json.loads(p.read_text()).keys())
    assert keys == base, f"{p.name}: missing {base-keys}, extra {keys-base}"
print("parity OK")
PY
```
